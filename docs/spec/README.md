# Spec

This directory holds the normative Steerable App specification: the contract for capability declarations, autonomy policy, execution, context, ledger, external bridge design, and conformance.

Spec work is governed by the north-star's core abstraction, policy, and architecture sections ([§5](../../Steerable-Protocol-NorthStar.md#5-the-core-abstraction-capability-declarations), [§6](../../Steerable-Protocol-NorthStar.md#6-autonomy-and-policy), [§7](../../Steerable-Protocol-NorthStar.md#7-architecture)) and by [`ROADMAP.md#d2--spec-conventions-and-requirement-ids-normative-gap-fill`](../plan/ROADMAP.md#d2--spec-conventions-and-requirement-ids-normative-gap-fill).

Do not add empty stubs for future spec documents. A spec file belongs here only when it contains real normative or informative content for its issue.
